# Evolution of Todo - Hackathon II

## Project Navigation

This is a **monorepo** containing 5 phases of the Evolution of Todo hackathon project.

## Phases

| Phase | Directory | Description | Status |
|-------|-----------|-------------|--------|
| I | `phase-1-console/` | In-Memory Python Console App | **SEALED** |
| II | `phase-2-web/` | Full-Stack Web (Next.js + FastAPI + Neon) | Pending |
| III | `phase-3-ai/` | AI-Powered Chatbot (OpenAI Agents + MCP) | Pending |
| IV | `phase-4-k8s/` | Local Kubernetes (Minikube + Helm) | Pending |
| V | `phase-5-cloud/` | Advanced Cloud (Kafka + Dapr + DOKS) | Pending |

## Quick Start

### Phase 1 (Console App)
```bash
cd phase-1-console
uv sync
uv run python src/main.py
```

### Run All Tests
```bash
uv run pytest phase-1-console/tests/ -v
```

## Development Guidelines

See `.specify/memory/constitution.md` for code quality, testing, and architecture principles.

## Claude Code Rules

This project uses Spec-Driven Development (SDD):
1. **Specify** - Create feature specification
2. **Plan** - Design architecture with ADRs
3. **Tasks** - Break down into testable tasks
4. **Implement** - Code following specs
5. **PHR** - Record prompt history after each interaction
